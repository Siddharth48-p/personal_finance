# Ledger — Personal Expense Tracker

A clean full-stack expense tracker: manual entry + CSV import, auto-categorization,
budget limits with progress bars, and automatic recurring-payment detection.

**Stack:** FastAPI + SQLite (backend) · React + Vite + Tailwind + Recharts (frontend)

---

## Features

- Add transactions manually or bulk-import via CSV
- Auto-categorization by merchant keyword (Swiggy → Food & Dining, Netflix → Subscriptions, etc.)
- Monthly dashboard: income vs. expense, spend-by-category pie chart, top categories bar chart
- Set a budget limit per category — progress bar turns amber at 80%, red when over
- Recurring payment detection: flags merchants billed repeatedly with a similar amount across 2+ months

## Running locally (VS Code)

### 1. Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Backend runs at `http://127.0.0.1:8000`. Visit `http://127.0.0.1:8000/docs` for interactive API docs (auto-generated by FastAPI — great to show in an interview).

A SQLite file `ledger.db` is created automatically on first run.

### 2. Frontend

Open a **second terminal**:

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at `http://localhost:5173`. Open that in your browser.

### 3. Try it out

- Go to the **Transactions** tab → upload `sample_transactions.csv` (in the project root) to see auto-categorization and the dashboard populate instantly.
- Go to **Budgets** → set a limit on "Food & Dining" (e.g. ₹1000) → watch the progress bar.
- Go to **Recurring** → see Netflix, Rent, etc. auto-detected as recurring charges.

CSV format expected: `date,merchant,amount,note` — dates as `YYYY-MM-DD`, positive amount = expense, negative = income.

## Notes for interview talking points

- **Auto-categorization**: keyword-matching now, designed so it could be swapped for a trained classifier later without touching the API contract.
- **Recurring detection**: groups transactions by merchant, flags as recurring if seen in ≥2 distinct months with amounts within 10% of the mean — a simple statistical heuristic, not hardcoded rules per merchant.
- **Schema design**: `Category` and `Transaction` are separate normalized tables with a foreign key, and budget limits live on the category rather than being duplicated per transaction.
